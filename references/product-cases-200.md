# 产品案例库：200 个产品案例

本案例库整理了 200 个产品案例，用于寻找相似场景、练习识别真实需求，并辅助设计可验证的 3 小时 MVP。

- 成功案例：160 个
- 失败案例：40 个

每个案例保留公开来源链接。收入、团队、融资、失败原因等信息来自对应公开资料，应作为案例线索和类比材料，而不是独立审计事实。

## 使用方式

1. 先找和目标行业、用户角色、工作流最接近的 5 个案例。
2. 每个案例重点看 3 件事：需求背景、结果、启发。
3. 把相似案例改写成需求句：谁，在什么场景下，因为现有方案什么问题，付出了什么代价。
4. 将一个最值得验证的需求收束成 3 小时 MVP。

### 001. S002 dealpad

- **类型**：success / 垂直 SaaS / 工作流软件
- **需求背景**：B2B 软件公司的销售团队，在推进大客户成交时，需要把客户需求、销售材料、会议纪要、报价信息、下一步行动和内部协作都组织起来。过去这类流程常常靠销售个人经验、Excel 表格、CRM 备注、邮件来回沟通和临时文档拼接完成；问题是信息分散，销售方法难以复制，团队管理者也很难判断每个机会到底卡在哪里。销售付出的代价是：新人 ramp-up 慢，关键客户跟进不一致，优秀销售的方法沉淀不下来，最终直接影响转化率和成交金额。 Adam 在 Salesforce 工作期间，曾用 Excel 搭建过一套销售框架，并观察到它能明显帮助团队提升成交效率。后来他把这套原本依赖表格和个人经验的销售方法，产品化成 dealpad 这样的 Digital Sales Room 平台，帮助 B2B 软件组织把销售过程、客户协作和成交材料集中到一个可复制的空间里。这个产品先服务 B2B 销售团队这一窄场景，再通过案例和口碑扩展，公开结果达到约 \$20M/year，团队 12 人。
- **结果**：\$20M/year；启动成本：\$200K；团队：12
- **启发**：从一个窄场景切入，比一开始做大平台更容易验证。

### 002. S003 Tithely

- **类型**：success / 垂直 SaaS / 工作流软件

- **结果**：\$15.6M/year；团队：180

### 199. F048 Stayzilla

- **类型**：failure / 定位与执行失焦
- **需求背景**：印度旅行者需要更丰富、更本地化、更负担得起的住宿选择，尤其在酒店覆盖不足的城市和地区，民宿、客栈和家庭旅馆能补充供给。房东和小住宿业主也需要线上获客渠道。Stayzilla 提出做印度 homestay network，把非标住宿供给搬到线上，服务国内旅行需求。这个市场很大，但非标住宿平台需要同时解决供给审核、照片质量、定价、支付、客服、信任、安全和目的地密度。Stayzilla 早期经历多次方向变化，从酒店预订到民宿网络，最终因定位与执行失焦关闭。它说明大市场里必须找到清晰切口，不能在供给形态和商业模式之间反复摇摆。
- **结果**：Failory 记录的结果：Shut Down；开始时间：2005；结束/状态：2017；融资规模：\$10M-\$50M；失败标签：Lack of Focus。
- **启发**：失败案例先反查：需求、渠道、成本、执行哪一环断了。
- **来源**：[Stayzilla failure analysis](https://www.failory.com/cemetery/stayzilla)

### 200. F049 Auctionata

- **类型**：failure / 合规、信任或治理问题
- **需求背景**：艺术品和奢侈品买家希望更方便地参与拍卖，卖家也希望触达全球买家。传统拍卖行门槛高、地域限制强、流程不透明，线上化看起来能扩大市场。Auctionata 提出做艺术品与奢侈品在线拍卖平台，通过直播和线上竞拍让更多人参与高端收藏品交易。需求确实存在，但高价值拍卖极度依赖鉴定、 provenance、卖家诚信、竞拍公平、支付履约和监管合规。只要平台在信任或治理上出现漏洞，用户就会怀疑整个交易机制。Auctionata 最终因法律和治理挑战关闭，说明越高客单价、越依赖信任的市场，平台治理本身就是核心产品。
- **结果**：Failory 记录的结果：Shut Down；开始时间：2012；结束/状态：2017；融资规模：> \$50M；失败标签：Legal Challenges。
- **启发**：涉及交易、身份、内容和资金时，信任机制本身就是产品。
- **来源**：[Auctionata failure analysis](https://www.failory.com/cemetery/auctionata)
